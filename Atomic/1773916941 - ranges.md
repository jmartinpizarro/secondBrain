# ranges

A `range` is just a pair (begin, end) that returns iterators. Less verbose for iterating and delimiting the size of the strings/arrays/vectors...

```cpp
namespace rng = std::ranges;
std::array<int, 3> a {6, 2, 3};
auto isSix = [](int i) { return i == 6; };

// old way
auto it = std::find(a.begin(), a.end(), 6);

// new way
auto it = rng::find(a, isSix);
```

---

- range: something that can be iterated over
- range algorithm: algorithm that takes range
- view: lazy range that is cheap (to copy)
- range adaptor: make a range into a view
- views caveats
	- caching (filter) or owning views might not be cheap
	- also not safe to pass by const ref

## Range algorithms

Are like STL algorithms
- not always a drop in replacement
Execute immediately
- iteration is internally controlled by the algorithm
Added projection parameters
Return values improved

## Views and adaptors details

Views are ranges with 'lazy evaluation'.
- Non-owning (pointer and size of the element) of elements
- **All methods O(1)** -  copy and assignment
- Allows for composition of several processing steps
- Allows for 'infinite ranges'

```cpp
namespace rv = ranges:views;

std::vector<int> v1 {0, 1};
std::vector<int> v2 {2, 3};

for (auto&& [a, b]: rv::cartesian_product(v1, v2)) {
	std::print("{} {}\n", a, b);
}
```
